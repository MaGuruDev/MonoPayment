# MaGuru Monobank Payment — Посібник користувача

**Версія:** 1.1.2  
**Сумісність:** Adobe Commerce / Magento Open Source 2.4.4 – 2.4.x  
**PHP:** 8.1 або вище  
**Потребує:** MaGuru Monobank Core  
**Ліцензія:** Платно

---

## Огляд

MaGuru Monobank Payment інтегрує Monobank Acquiring як повноцінний спосіб оплати для вашого магазину Adobe Commerce. Підтримує стандартну оплату карткою, авторизацію з утриманням (Hold), збережені картки, Apple Pay / Google Pay через MonoPay Button, оплату через QR-термінал, розділені платежі для маркетплейсів, автоматичні рефанди, надсилання фіскального чека та інше.

### Огляд можливостей

| Можливість | Опис |
|---|---|
| **Стандартна оплата карткою** | Оплата через redirect або вбудований iFrame з формою Monobank |
| **Hold / Capture** | Платежі лише з авторизацією: утримання коштів, списання при відвантаженні |
| **Збережені картки** | Клієнти зберігають картки для покупки в один клік наступного разу |
| **MonoPay Button** | Кнопка швидкої оплати Apple Pay / Google Pay на чекауті |
| **Оплата через QR-термінал** | Клієнт сканує QR-код Monobank для оплати |
| **Розділені платежі (Split)** | Направлення частини платежу на рахунок субмерчанта |
| **Автоматичні рефанди** | Повернення через Monobank при створенні Credit Memo в Magento |
| **Надсилання квитанції** | Лист клієнту з квитанцією про оплату від Monobank |
| **Курс валют** | Отримує курс гривні від Monobank кожні 4 години |
| **Синхронізація виписки банку** | Щоденна синхронізація виписки еквайрингу; грід показує прибуток |
| **Сповіщення про реверс** | Лист клієнту, коли Monobank скасовує (реверсить) платіж |

### Налаштування черги повідомлень (обов'язково)

MonoPayment обробляє кілька операцій асинхронно — авто-списання Hold при відвантаженні, скасування/видалення інвойсу, надсилання квитанції. Ці операції йдуть через чергу повідомлень Magento і **нічого не роблять, доки не запущено consumer**; сам Magento не запускає consumer'и автоматично.

Додайте наступне в `app/etc/env.php`:

```php
'cron_consumers_runner' => [
    'cron_run' => true,
    'max_messages' => 200,
    'consumers' => [
        'mono_payment.invoice.cancel',
        'mono_payment.invoice.finalize',
        'mono_payment.invoice.remove',
        'mono_payment.invoice.receipt',
    ]
],
```

Далі cron самого Magento автоматично запускатиме ці consumer'и — жодного додаткового процесу не потрібно. Якщо цей крок пропустити, замовлення з Hold ніколи не спишуться автоматично при відвантаженні, а листи-квитанції ніколи не надішлються, без жодної помилки де-небудь.

## Налаштування

Перейдіть у **Stores → Configuration → Sales → Payment Methods → Monobank Payment**.

> **API-креденшали живуть в іншому місці.** **API Token** Monobank, кнопка **Validate Token**, примітка **Payment Webhook URL** і поле **API Base URL (Dev Override)** налаштовуються в **Stores → Configuration → MaGuru → Monobank Integration → Acquiring API** — не на цій сторінці способу оплати. Деталі про цю секцію — у посібнику користувача *Monobank Core*.

### Секція 1: General (Загальні)

| Параметр | Опис | За замовчуванням |
|---|---|---|
| **Enabled** | Активує спосіб оплати на чекауті | No |
| **Title** | Назва, яку бачить клієнт на чекауті (напр. "Pay with Monobank") | Monobank (Visa/Mastercard, Apple Pay, Google Pay) |
| **Checkout Button Label** | Текст на кнопці підтвердження оплати на чекауті | Pay via Monobank |
| **Payment Type** | `debit` — списання одразу; `hold` — лише авторизація, списання при відвантаженні | debit |
| **Checkout Display Mode** | `redirect` — клієнт покидає магазин для оплати; `iframe` — форма оплати вбудована на сторінці | redirect |
| **Invoice Validity (seconds)** | Скільки секунд інвойс Monobank лишається дійсним, перш ніж стане прострочений | 3600 (1 година) |
| **New Order Status** | Статус замовлення, що призначається новим замовленням, оплаченим через Monobank | Pending |
| **Debug Logging** | Записувати детальні дані запитів/відповідей платежу в лог | No |

### Секція 2: Saved Cards (Збережені картки)

| Параметр | Опис | За замовчуванням |
|---|---|---|
| **Save Cards (Tokenization)** | Дозволяє авторизованим клієнтам зберігати картку під час чекауту. Потребує активації Wallet/Tokenization з боку служби підтримки Monobank | No |
| **Enable MonoPay Button (JS Widget)** | Показує кнопку Apple Pay / Google Pay на чекауті | No |
| **Generate & Register Key** | Кнопка, що генерує нову пару ключів ECDSA P-256, реєструє публічний ключ у Monobank і автоматично заповнює поля Key ID та Private Key | — |
| **MonoPay Button Key ID** | ID публічного ECDSA-ключа від Monobank (обов'язково для MonoPay Button) | — |
| **MonoPay Button Private Key (PEM)** | Ваш приватний ECDSA P-256-ключ (зберігається зашифровано; обов'язково для MonoPay Button) | — |

> **Налаштування MonoPay Button:** використайте кнопку **Generate & Register Key** вище, щоб автоматично згенерувати й зареєструвати пару ключів ECDSA. Альтернативно, якщо приватний ключ уже налаштований вручну, запустіть CLI-команду `bin/magento mono:monopay-button:setup`, щоб отримати публічний ключ і завершити реєстрацію в Monobank.

### Секція 3: QR Terminal (QR-термінал)

| Параметр | Опис | За замовчуванням |
|---|---|---|
| **Enable QR Payment** | Активує оплату через QR-термінал на чекауті | No |
| **QR Terminal ID** | ID терміналу з налаштувань вашого QR-термінала Monobank | — |
| **QR Amount Type** | Як термінал обробляє суму платежу. **Merchant sets amount** (`amountType: merchant`) — сума замовлення надсилається в інвойсі; працює зі стандартним API-чекаутом. **Client/Fixed** (`amountType: client` або `fix`) — клієнт вводить суму на терміналі або вона фіксована; сума в API-запиті не надсилається. ## Адмін-панелі

### Monobank Merchant (Мерчант)

**Monobank → Merchant**

Показує дані облікового запису мерчанта, отримані наживо з Monobank Acquiring (напр. назву мерчанта та MCC).

---

### Monobank Invoices (Інвойси)

**Monobank → Invoices**

Грід, що показує всі платіжні інвойси Monobank, створені вашим магазином.

| Колонка | Опис |
|---|---|
| **Invoice ID** | Внутрішній ідентифікатор інвойса Monobank |
| **Order #** | Номер замовлення Magento (increment ID) |
| **Status** | Поточний статус платежу (див. таблицю нижче) |
| **Amount (kopecks)** | Сума інвойса, відображається відформатованою в UAH |
| **Payment Type** | debit або hold |
| **Payment Method** | Платіжна система картки / гаманець, використаний для оплати (якщо доступно) |
| **Masked PAN** | Останні 4 цифри картки, якщо доступно |
| **Created At** | Час створення інвойса |
| **Updated At** | Час останнього оновлення статусу |

#### Статуси інвойса

| Статус | Значення |
|---|---|
| **Created** | Інвойс створено в Monobank, очікується оплата |
| **Processing** | Оплата ініційована, очікується дія або підтвердження клієнта |
| **Success** | Оплата успішно завершена |
| **Hold** | Сума авторизована й утримана, очікує списання |
| **Reversed** | Оплату скасовано (реверснуто) або повернено Monobank |
| **Failure** | Оплата не пройшла або відхилена |
| **Expired** | Інвойс прострочений без оплати |

## Перегляд замовлення в адмінці

Коли замовлення оплачене через Monobank, на сторінці перегляду замовлення з'являється блок **Monobank Payment**:

- Invoice ID, статус, сума, тип платежу, замаскований номер картки
- Кнопка **Sync Status** — миттєво отримує актуальний статус від Monobank
- Кнопка **Send Receipt** — повторно надсилає клієнту квитанцію про оплату Monobank (видима лише для успішних платежів)
- Кнопка **Capture Payment** — фіналізує Hold-платіж (видима лише для платежів типу Hold)

---

## Особистий кабінет клієнта: My Cards (Мої картки)

Коли увімкнено Saved Cards, авторизовані клієнти бачать секцію **My Cards** у своєму кабінеті. Вони можуть:

- Переглядати збережені картки (показуються як замаскований номер)
- Видаляти збережені картки
- Обирати збережену картку під час чекауту, щоб оплатити без повторного введення даних картки

## Флоу Hold / Capture

Коли **Payment Type** встановлено на `hold`:

1. На чекауті сума авторизується (утримується) на картці клієнта — списання ще не відбувається
2. Замовлення переходить у стан **Payment Review**; статус інвойса Monobank показує **Hold**
3. На цьому етапі інвойс Magento не створюється
4. Щоб списати (стягнути утримані кошти):
   - **Автоматично при відвантаженні** — коли ви створюєте відвантаження в Magento, Hold списується автоматично
   - **Вручну** — використайте кнопку **Capture Payment** у перегляді замовлення в адмінці
5. Після успішного списання Monobank підтверджує через webhook (`success`), і інвойс Magento створюється автоматично

> Якщо не списати кошти протягом вікна авторизації Monobank (зазвичай 9 днів для типу `hold`), утримання скасовується і списання не відбувається.

## Рефанди

Щоб повернути кошти по платежу Monobank:

1. Відкрийте замовлення в адмінці Magento
2. Створіть **Credit Memo** як зазвичай
3. Модуль автоматично надсилає запит на рефанд у Monobank на суму Credit Memo

Жодних ручних дій у Monobank не потрібно — рефанд запускається самим процесом створення Credit Memo в Magento.

## Часті запитання

**Q: Як увімкнути платежі Hold / Capture?**

Встановіть **Payment Type** на `hold` в конфігурації оплати. Платіж буде утриманий на картці клієнта під час чекауту й автоматично списаний при створенні відвантаження. Також можна списати вручну з перегляду замовлення кнопкою **Capture Payment**.

---

**Q: Клієнт хоче рефанд. Як це обробити?**

Створіть стандартний Credit Memo Magento для замовлення. Модуль автоматично надсилає рефанд у Monobank — окремої дії в кабінеті Monobank не потрібно.

---

**Q: Статус інвойса застряг на "Processing". Як це виправити?**

Використайте кнопку **Sync Status** у перегляді замовлення в адмінці, щоб миттєво отримати актуальний статус від Monobank. Або запустіть з командного рядка `bin/magento mono:payment:status-check --invoice-id=X`.

---

**Q: Як налаштувати MonoPay Button (Apple Pay / Google Pay)?**

1. У **Stores → Configuration → Sales → Payment Methods → Monobank Payment → Saved Cards** увімкніть **Enable MonoPay Button (JS Widget)**
2. Натисніть **Generate & Register Key**, щоб згенерувати пару ключів ECDSA P-256 і автоматично зареєструвати її в Monobank
3. Альтернативно, якщо приватний ключ уже налаштований вручну, запустіть `bin/magento mono:monopay-button:setup`, щоб отримати публічний ключ і зареєструвати його в Monobank

---

**Q: Які валюти підтримуються?**

Monobank Acquiring працює лише в гривні (UAH). Для магазинів з валютами відображення, відмінними від UAH, провайдер курсу валют отримує актуальний курс з публічного API Monobank кожні 4 години, щоб товари могли відображатись в інших валютах, але фактичне списання завжди відбувається в гривні.

---

**Q: Платіж клієнта був реверснутий Monobank. Чи отримає він сповіщення?**

Так. Коли Monobank надсилає webhook про реверс, модуль оновлює статус інвойса на **Reversed** і надсилає клієнту email-сповіщення.

---

## Підтримка

Перед зверненням у підтримку зберіть:

1. Invoice ID Monobank і номер замовлення (Order #) з гріда Invoices
2. Статус інвойса та будь-яку помилку, видиму в адмінці
3. Вміст `var/log/mono_payment.log`, якщо увімкнено Debug Logging
4. Орієнтовний час виникнення проблеми
